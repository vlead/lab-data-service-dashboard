1 Introduction

This document describes all the requirements of lab-data-service-dashboard.

2 Requirements

2.1 Req1

  • Ability to do all the CRUD operations on labs data based on the given role and a User.

2.2 Req2

  • Admin Role should be able to perform all the CRUD operations.
  • Admin Role should be able to add other users and assign roles to them.

2.3 Req3

  • Guest Role should be able to Read/View the labs data.

2.4 Req4

  • Authentication and Authorization should be provided for all the LDS Dashboard users.

2.5 Req5

  • A summary dashboard of LDS is the landing page of this application.
  • The dashboard contains total number of labs in phase2, labs hosted on AWS, labs hosted at institute servers, labs on Open edX, etc.
